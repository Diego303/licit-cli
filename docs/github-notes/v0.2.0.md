# licit v0.2.0 — Provenance

> Code provenance tracking is here. licit can now analyze your git history and AI agent session logs to identify what code was written by AI and what by humans — with cryptographic attestation.

## Highlights

- **Full provenance pipeline**: git analysis → heuristic scoring → classification → signed storage → Markdown reporting
- **6-heuristic AI detection engine** with weighted scoring (author patterns, commit messages, bulk changes, co-authors, file patterns, time patterns)
- **Claude Code session reader** extracts provenance from agent session logs
- **Cryptographic attestation**: HMAC-SHA256 individual signing + Merkle tree batch verification
- **280 tests** passing — mypy strict clean, ruff clean
- **4 functional CLI commands**: `init`, `status`, `connect`, `trace`

## What's new

### `licit trace` command (fully functional)

```bash
licit trace                      # Analyze full git history
licit trace --since 2026-01-01   # From a specific date
licit trace --stats              # Show statistics in terminal
licit trace --report             # Generate Markdown report
```

**Example output:**

### Heuristic engine

6 independent heuristics score each commit. Only signaling heuristics (score > 0) contribute to the weighted average:

| Heuristic | Weight | Detects |
|---|---|---|
| Author pattern | 3.0 | AI author names (claude, copilot, cursor, bot, etc.) |
| Message pattern | 1.5 | AI commit patterns (conventional, `[ai]`, `implement`, etc.) |
| Bulk changes | 2.0 | Massive changes (>20 files and >500 lines) |
| Co-author | 3.0 | `Co-authored-by:` with AI keywords |
| File patterns | 1.0 | All files are test files |
| Time pattern | 0.5 | Commits between 1-5 AM |

Classification: `ai` (score >= 0.7), `mixed` (>= 0.5), `human` (< 0.5).

### Session readers

Protocol-based architecture for reading AI agent session logs. v0.2.0 ships with a Claude Code reader:

```yaml
provenance:
  methods:
    - git-infer
    - session-log
  session_dirs:
    - ~/.claude/projects/
```

### Cryptographic attestation

Optional HMAC-SHA256 signing with Merkle tree batch verification:

```yaml
provenance:
  sign: true
  sign_key_path: ~/.licit/signing-key
```

Key management: explicit path → `.licit/.signing-key` fallback → auto-generation (32 bytes).

### Provenance store

Append-only JSONL store with load, count, clear, and append operations. Each record includes file path, source classification, confidence score, detection method, timestamp, model, and agent tool.

### Markdown reports

```bash
licit trace --report
# Generates .licit/reports/provenance.md
```

Report includes summary statistics, per-file classification table, detected tools and models.

## QA hardening

9 bugs found and fixed during QA, including:

- **Bug #4 (critical)**: `ValueError: embedded null byte` in git_analyzer.py — literal null bytes in subprocess args replaced with git hex escapes (`%x00`/`%x01`). Invisible in unit tests (mocked subprocess), caught during E2E integration test.
- Heuristic scoring fixes for edge cases (Unicode authors, empty commits, boundary scores)
- Store serialization fixes for missing fields and special characters
- Attestation key file error handling

81 additional edge-case tests written during QA.
