# licit v0.3.0 — Changelog

> Agent config change tracking is here. licit now monitors changes to AI agent configuration files (CLAUDE.md, .cursorrules, AGENTS.md, etc.) across git history — with semantic diffing, severity classification, and Markdown/JSON output.

## Highlights

- **Full changelog pipeline**: git history watcher → semantic differ → severity classifier → Markdown/JSON renderer
- **Format-aware semantic diffing**: YAML (recursive key-value), JSON (recursive key-value), Markdown (section-aware with code block tracking), plain text (whole-file)
- **Severity classification**: MAJOR (model/provider changes), MINOR (prompt/guardrail/tools), PATCH (everything else), with removal escalation
- **373 tests** passing — mypy strict clean, ruff clean
- **5 functional CLI commands**: `init`, `status`, `connect`, `trace`, `changelog`

## What's new

### `licit changelog` command (fully functional)

```bash
licit changelog                        # Generate Markdown changelog
licit changelog --format json          # JSON output
licit changelog --since 2026-01-01     # Since specific date
```

**Example output:**
```markdown
# Agent Config Changelog

> 3 change(s) detected across 2 file(s): **1** major, **1** minor, **1** patch

## .architect/config.yaml

- **[MAJOR]** Changed: model from claude-sonnet-4 to claude-opus-4 (`a1b2c3d4`) — 2026-03-12
- **[PATCH]** Changed: budget.max_cost_usd from 5.0 to 10.0 (`a1b2c3d4`) — 2026-03-12

## CLAUDE.md

- **[MINOR]** Changed: section:Rules from 5 lines to 8 lines (+3/-0) (`e5f6g7h8`) — 2026-03-11
```

### Semantic differ

Format-aware diffing produces meaningful field-level diffs instead of raw line diffs:

| Format | Strategy |
|---|---|
| YAML (`.yaml`, `.yml`) | Recursive key-value diff, nested dict support |
| JSON (`.json`) | Recursive key-value diff |
| Markdown (`.md`) | Section-aware diff by headings (with fenced code block awareness) |
| Plain text | Whole-file line diff with added/removed counts |

Non-dict YAML/JSON roots (lists, scalars) are wrapped as `{"(root)": data}` instead of being silently dropped.

### Severity classification

Segment-based field matching prevents false positives (`model_config` does NOT match pattern `model`):

| Severity | Trigger | Examples |
|---|---|---|
| **MAJOR** | Model/provider change, or removal of a MINOR field | `model: gpt-4` → `gpt-5`, deleting `guardrails` |
| **MINOR** | Prompt, guardrails, tools, rules, markdown section changes | Editing `system_prompt`, adding `blocked_commands` |
| **PATCH** | Everything else | Parameter tweaks, formatting |

### Changelog renderer

- **Markdown**: Grouped by file, sorted by severity (MAJOR first) then timestamp descending, summary counts, per-file sections, UTC footer
- **JSON**: Structured `{"changes": [...]}` records with `ensure_ascii=False` for full Unicode support

### Config watcher

Monitors agent config files across git history:
- Uses `git log --follow` for file rename tracking
- Size guard: 1 MB limit on `git show` output (prevents memory exhaustion from accidentally tracked binaries)
- All subprocess calls with explicit timeouts (10s/30s) and `check=False`
- Glob pattern support in `watch_files` (e.g., `.prompts/**/*.md`)

## New modules

| Module | Purpose |
|---|---|
| `changelog/watcher.py` | Git history tracking of config files |
| `changelog/differ.py` | Semantic diff engine (YAML/JSON/MD/text) |

## QA hardening

7 bugs found and fixed during QA:

- **(High)** Substring matching (`"model" in field`) caused false MAJOR classifications on fields like `model_config` — replaced with segment-based `_field_matches()`
- **(High)** Non-dict YAML/JSON roots silently dropped — replaced with `_coerce_to_dict()` wrapping
- **(High)** `git show` without size limit could load GBs of binary content — added 1 MB guard
- **(High)** `datetime.now()` without timezone created naive timestamps that could crash the renderer sort — fixed to `datetime.now(tz=UTC)`
- **(Medium)** CLI file write without error handling — wrapped in `try/except OSError`
- **(Medium)** Watcher didn't log on timeout/failure — added `logger.debug`
- **(Low)** Markdown headings inside fenced code blocks parsed as real sections — added `in_code_block` tracking

93 changelog tests written (66 module tests + 27 QA edge cases).
