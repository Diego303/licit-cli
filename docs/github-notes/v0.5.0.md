# licit v0.5.0 — OWASP Agentic Top 10

> licit now evaluates your project against both the EU AI Act **and** the OWASP Top 10 for Agentic AI Security. 10 risk categories evaluated with scored evidence, actionable recommendations, and full CI/CD gate support.

## Highlights

- **OWASP Agentic Top 10 evaluator**: 10 risks evaluated with per-risk scoring (ASI01–ASI10)
- **Dual-framework verification**: `licit verify` now evaluates EU AI Act (11 articles) + OWASP (10 risks) = 21 total controls
- **600 tests** passing — mypy strict clean, ruff clean
- **8 functional CLI commands**: `init`, `status`, `connect`, `trace`, `changelog`, `fria`, `annex-iv`, `verify`

## What's new

### `licit verify --framework owasp` (fully functional)

```bash
licit verify --framework owasp       # Evaluate OWASP only
licit verify --framework all         # Evaluate EU AI Act + OWASP
licit verify                         # Same as --framework all
```

**Example output:**
```
  Compliance Verification
  Compliant:     1
  Partial:       7
  Non-compliant: 2

  Non-compliant controls:
    [X] ASI06: Lack of Human Oversight
    [X] ASI08: Unbounded Resource Consumption
```

Exit codes: `0` = all compliant, `1` = non-compliant, `2` = partially compliant.

### OWASP Agentic Top 10 — risk-by-risk scoring

Each risk has a dedicated evaluation method with numeric scoring:

| Risk | What it checks | Compliant at |
|---|---|---|
| ASI01 — Excessive Agency | Guardrails, quality gates, budget limits, agent configs | 3+ of 4 |
| ASI02 — Prompt Injection | vigil scanning (+2), guardrails, human review | 3+ of 4 |
| ASI03 — Supply Chain | SCA tools (Snyk/Semgrep/CodeQL/Trivy) (+2), changelog, versioned configs | 3+ of 4 |
| ASI04 — Logging & Monitoring | Git history, audit trail (+2), provenance, OTel | 3+ of 5 |
| ASI05 — Output Handling | Human review (+2), quality gates, test suite | 3+ of 4 |
| ASI06 — Human Oversight | Human review (+2), dry-run, quality gates, rollback | 3+ of 5 |
| ASI07 — Sandboxing | Guardrails (+2), CI/CD isolation, agent scope | 3+ of 4 |
| ASI08 — Resource Consumption | Budget limits (+2), quality gates | 2+ of 3 |
| ASI09 — Error Handling | Test suite, CI/CD, rollback | 2+ of 3 |
| ASI10 — Data Exposure | Guardrails, security scanning (+2), agent scope | 3+ of 4 |

Each non-compliant risk includes **actionable recommendations** with specific tools and commands to close the gap.

### Smart evidence messages

The evaluator produces context-aware evidence strings:

- `has_guardrails=true, count=10` → "Guardrails limit agent scope: 10 rules"
- `has_guardrails=true, count=0` → "Guardrails configured (no specific rules counted)"
- Multiple SCA tools → "Dependency scanning: Snyk, Semgrep, CodeQL, Trivy"
- CI/CD platform → "CI/CD (github-actions) provides isolated execution environment"

## QA hardening

3 issues found and fixed during QA:

- **(Medium)** Report template format was inconsistent with EU AI Act template (different heading levels, `status.value` vs `status`, em dash vs parentheses) — aligned to match exactly
- **(Low)** `_safe_float()` hardcoded `field="ai_percentage"` in debug log despite generic name — added `field` keyword parameter
- **(Low)** `has_guardrails=True` with `guardrail_count=0` produced "0 rules" — added conditional branch showing "configured" instead

103 Phase 5 tests written:
- `test_evaluator.py` (40) — Properties, full evaluation, ASI01–ASI10 compliant/partial/non-compliant paths
- `test_requirements.py` (15) — Data integrity, unique IDs, categories, helpers
- `test_qa_edge_cases.py` (48) — Protocol conformance, `_score_to_status` boundaries, `_safe_float` robustness, idempotency, compliant→empty recs, non-compliant→has recs, security_findings neutrality, guardrail_count=0, template rendering, registry interop, CLI integration, cross-module compatibility

Test infrastructure: `conftest.py` `make_evidence()` now exposes 100% of `EvidenceBundle` fields (added `has_otel`, `has_requirements_traceability`, `security_findings_*`).
