# licit v0.1.0 — Foundation

> First public release. Establishes the project foundation: configuration system, auto-detection engine, evidence collection, CLI skeleton, and comprehensive test suite.

## Highlights

- **10 CLI commands** registered — 3 fully functional (`init`, `status`, `connect`), 7 ready for future phases
- **Auto-detection** of languages, frameworks, CI/CD, AI agent configs, security tools, and git history
- **Evidence collection** from project configs, architect guardrails, SARIF files, and `.licit/` data
- **113 tests** passing — mypy strict clean, ruff clean
- **Full documentation** in Spanish (10 documents covering architecture, CLI, config, security, compliance, and more)

## What's included

### Configuration system

- Pydantic v2 schema with 9 config models and sensible defaults
- YAML config loader with 3-level resolution: explicit path → `.licit.yaml` in cwd → defaults
- Validation including `confidence_threshold` bounded to [0.0, 1.0]

### Core data models

- 3 enums: `ComplianceStatus`, `ChangeSeverity`, `ProvenanceSource` (Python 3.12 `StrEnum`)
- 6 dataclasses: `ProvenanceRecord`, `ConfigChange`, `ControlRequirement`, `ControlResult`, `ComplianceSummary`, `GapItem`

### Project auto-detection

`ProjectDetector` with 8 detection methods:

| Detection | Sources |
|---|---|
| Project name | `pyproject.toml`, `package.json` |
| Languages | Python, JavaScript, TypeScript, Go, Rust, Java |
| Frameworks | FastAPI, Flask, Django, React, Next.js, Express |
| Agent configs | CLAUDE.md, .cursorrules, AGENTS.md, copilot-instructions, architect, .prompts/ |
| CI/CD | GitHub Actions, GitLab CI, Jenkins, CircleCI |
| Testing | pytest, jest, vitest |
| Security tools | vigil, semgrep, snyk, CodeQL |
| Git | commits, contributors |

### Evidence collection

`EvidenceCollector` gathers compliance evidence from:
- `.licit/` directory (provenance, FRIA, Annex IV, changelog)
- Architect config (guardrails, quality gates, budget limits, dry-run, rollback)
- Architect reports (audit trail)
- SARIF files (vigil security findings by severity)
- CI/CD config (human review gate detection)

```bash

```

```bash

cd licit-cli
pip install -e ".[dev]"
```
