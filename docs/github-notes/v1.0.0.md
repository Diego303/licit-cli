# licit v1.0.0 — Stable Release

> First stable release. All 10 CLI commands battle-tested across 142 manual tests, 5 simulated projects, and 10 edge case categories. Critical bugs fixed, store architecture improved, and new `--auto` mode for CI/CD pipelines.

## Highlights

- **Exhaustive QA**: 142 tests across 5 project types (Python/FastAPI, Node/Express, Rust, empty repo, architect+vigil) with all command/flag combinations
- **`licit fria --auto`**: Non-interactive FRIA for CI/CD — accepts auto-detected values and defaults without requiring TTY
- **Provenance store redesigned**: merge + deduplicate replaces append-only — store size proportional to unique files, not number of runs
- **`trace --since` fixed**: Now correctly filters by author date (was broken due to git committer date vs author date mismatch)
- **PermissionError handled gracefully**: No more raw Python tracebacks — clean error messages for all I/O errors
- **Config error visibility**: Corrupt/invalid YAML now shows visible warning instead of silent fallback
- **789 tests** passing — mypy strict clean, ruff clean, 0 known bugs

## Breaking changes

None. All V0 commands, config fields, data formats, and exit codes are preserved.

### Behavioral changes (non-breaking)

- **Provenance store** is now deduplicated on write instead of append-only. Running `licit trace` multiple times no longer grows the store file. The store always contains exactly one record per unique file.
- **`changelog --format json`** now saves to `.licit/changelog.json` instead of overwriting `.licit/changelog.md` with JSON content.
- **`licit init`** now warns when overwriting existing configuration.
- **`trace` output** now shows "Analyzed N files across M records" instead of "Analyzed M file records" for clarity.

## What's fixed

### Critical

| Bug | Impact | Fix |
|---|---|---|
| `trace --since` did not filter commits | Date filtering completely broken for projects with `--date=` overrides, cherry-picks, rebases | Moved filtering from git `--since` (committer date) to Python `_filter_since()` (author date) |
| `trace` crashed with traceback on PermissionError | Raw Python traceback shown to users in CI/CD or read-only environments | `store.save()` catches `OSError` → clean `click.ClickException` message |

### High

| Bug | Impact | Fix |
|---|---|---|
| Store grew unboundedly (N runs × M records) | 10 trace runs on a 37-file project = 370 lines | `save()` merges + deduplicates by file path (latest timestamp wins), rewrites atomically |
| `trace` vs `--stats` showed different numbers | "37 files" vs "35 files" — confusing for users | `trace` deduplicates for display, shows "N files across M records" |

### Medium

| Bug | Impact | Fix |
|---|---|---|
| `changelog --format json` saved JSON in `.md` file | Markdown changelog overwritten with JSON | Output extension matches format (`.json` for JSON) |
| `reports.output_dir` config ignored | Reports always saved to `.licit/reports/` | CLI now uses `config.reports.output_dir` |
| `init` silently overwrote existing config | Risk of losing custom configuration | Warning displayed on re-initialization |
| `gaps` said "No gaps found!" with no frameworks enabled | Misleading — nothing was evaluated | Shows "No frameworks enabled" message |

### Low

| Bug | Impact | Fix |
|---|---|---|
| Config errors only visible with `--verbose` | Users unaware of corrupt YAML | `click.echo` warning to stderr |

## What's new

### `licit fria --auto`

Non-interactive mode for CI/CD pipelines:

```bash
licit fria --auto
```

- Accepts all auto-detected values (system purpose, AI technology, testing, audit trail, etc.)
- Uses first-choice defaults for unanswered questions
- Generates both `fria-data.json` and `fria-report.md`
- Empty fields can be filled later with `licit fria --update`

**CI/CD example:**
```yaml
- name: Generate FRIA
  run: licit fria --auto

- name: Generate Annex IV
  run: licit annex-iv

- name: Verify compliance
  run: licit verify
```

## Complete CI/CD pipeline

```yaml
name: Compliance Pipeline
on:
  pull_request:
    branches: [main]

jobs:
  compliance:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'

      - name: Install licit
        run: pip install licit-ai-cli

      - name: Initialize
        run: licit init

      - name: Track provenance
        run: licit trace

      - name: Generate FRIA (non-interactive)
        run: licit fria --auto

      - name: Generate Annex IV
        run: licit annex-iv

      - name: Generate report
        run: licit report --format html -o compliance-report.html

      - name: Upload report
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: compliance-report
          path: compliance-report.html

      - name: Verify compliance (gate)
        run: licit verify
```
