# licit v0.7.0 — Connectors + Integration

> licit now has **formal connectors** for architect and vigil that read reports, audit logs, SARIF security findings, and SBOM data to enrich compliance evidence. The **V0 MVP is feature-complete** — all 7 phases implemented, all 10 CLI commands fully functional.

## Highlights

- **Architect connector**: reads JSON reports, JSONL audit logs, and YAML config for guardrails, quality gates, budget limits, dry-run/rollback capabilities
- **Vigil connector**: parses SARIF 2.1.0 files from any security scanner (vigil, Semgrep, CodeQL) and CycloneDX SBOM
- **Connector protocol**: `Connector` Protocol with `available()`, `collect()`, and computed `ConnectorResult.success`
- **Zero duplication**: inline evidence paths delegate to connector classes instead of reimplementing parsing logic
- **Enhanced CLI**: `licit connect` shows data availability, `licit status` shows security findings count
- **End-to-end integration tests**: full flow from `init` to `verify` on synthetic git projects
- **789 tests** passing — mypy strict clean, ruff clean
- **V0 MVP complete**: all 7 implementation phases finished

## What's new

### Architect connector

```bash
licit connect architect
# → architect data found at: .architect/reports
# → Connector 'architect' enabled.
```

Reads three data sources:

| Source | Format | Evidence |
|---|---|---|
| Reports | `.architect/reports/*.json` | Audit trail entries |
| Audit log | `.architect/audit.jsonl` | Audit trail (task lifecycle events) |
| Config | `.architect/config.yaml` | Guardrails, quality gates, budget, dry-run, rollback |

**Config fields extracted:**
```yaml
guardrails:
  protected_files: [.env]     # → guardrail_count += N
  blocked_commands: [rm -rf]  # → guardrail_count += N
  code_rules: [no-eval]       # → guardrail_count += N
  quality_gates: [lint, test] # → has_quality_gates
costs:
  budget_usd: 50              # → has_budget_limits
```

### Vigil connector

```bash
licit connect vigil --enable
# → vigil data found
# → Connector 'vigil' enabled.
```

Parses SARIF 2.1.0 from **any** security scanner:

| SARIF level | Classification |
|---|---|
| `error` | `security_findings_critical` |
| `warning` | `security_findings_high` |
| `note` | counted in total |

**SARIF path resolution**: explicit file, directory (`*.sarif` glob), or auto-detected from project — with deduplication.

**SBOM support**: reads CycloneDX JSON, validates structure (V1: will feed into OWASP ASI03 evaluation).

### Enhanced CLI

**`licit status`** now shows connector state and security findings:
```
  Connectors:
    [x] architect (.architect/config.yaml, enabled)
    [x] vigil (.vigil.yaml, enabled)
    Security findings: 3 total (1 critical, 1 high)
```

**`licit connect`** now verifies data availability:
```
  architect data found at: .architect/reports
  Connector 'architect' enabled.
```

### EvidenceCollector refactored

- Accepts optional `LicitConfig` for formal connector support
- When config provided and connector enabled → delegates to connector class
- When no config → constructs temporary connector with auto-detected paths
- Zero code duplication between formal and inline paths

## Quick start

```bash
cd your-project
licit init
licit connect architect              # Enable architect connector
licit trace --stats                  # Track provenance
licit report --format html -o r.html # Compliance report
licit gaps                           # What's missing?
licit verify                         # CI/CD gate
```
