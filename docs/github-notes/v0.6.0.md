# licit v0.6.0 — Reports + Gap Analyzer

> licit now generates **unified compliance reports** in three formats (Markdown, JSON, HTML) and identifies **compliance gaps** with actionable recommendations, tool suggestions, and effort estimates. All 10 CLI commands are fully functional.

## Highlights

- **Unified compliance reports**: Multi-framework reports with overall statistics, per-framework breakdown, and per-requirement detail
- **Three output formats**: Markdown (tables + icons), JSON (structured), HTML (self-contained, responsive, color-coded badges)
- **Gap analyzer**: Identifies non-compliant and partial requirements, sorted by severity, with specific tool recommendations and effort estimates
- **Terminal summary**: Progress bars and compliance rate at a glance
- **706 tests** passing — mypy strict clean, ruff clean
- **10 functional CLI commands**: all commands now fully operational

## What's new

### `licit report` (fully functional)

```bash
licit report                              # Markdown (default)
licit report --framework eu-ai-act        # EU AI Act only
licit report --format json -o report.json # JSON output
licit report --format html -o report.html # Self-contained HTML
```

### `licit gaps` (fully functional)

```bash
licit gaps                           # All frameworks
licit gaps --framework eu-ai-act     # EU AI Act only
licit gaps --framework owasp         # OWASP only
```

**Example output:**
```
  10 compliance gap(s) found:

  1. [X] [ART-27-1] Fundamental Rights Impact Assessment (FRIA)
     Missing: Before putting an AI system into use, deployers shall
     carry out an assessment of the impact on fundamental rights.
     -> Run: licit fria -- to complete the FRIA
     Tools: licit fria

  2. [X] [ASI02] Prompt Injection
     Missing: AI agents must be protected against adversarial
     manipulation through crafted inputs.
     -> Add AI-specific security scanning
     Tools: vigil (prompt injection scanner)
```

Each gap includes:
- **Status icon**: `[X]` non-compliant, `[!]` partial
- **Description**: What's missing or incomplete, with evidence
- **Recommendation**: Specific action to close the gap
- **Tools**: Concrete tools/commands to use
- **Effort**: `low`, `medium`, or `high`

## QA hardening

7 issues found and fixed during QA:

- **(Critical)** Gap analyzer used wrong OWASP category names — 8 of 10 categories didn't match `requirements.py` (e.g., `excessive-agency` vs actual `access-control`). Result: all OWASP gaps had empty tool suggestions. Fixed by cross-referencing actual category keys.
- **(High)** No exception handling in `fw.evaluate()` calls — a crashing evaluator would take down the entire report. Fixed with `try/except` + `logger.exception()`.
- **(High)** No CLI integration tests for `report`, `gaps`, `verify`. Added 10 tests.
- **(Medium)** HTML `_esc()` didn't escape single quotes (`'`). Fixed with `&#39;`.
- **(Medium)** `data-governance` had empty tool suggestions. Fixed.
- **(Low)** Dead `structlog` imports in renderers. Removed.
- **(Low)** `json_fmt._framework_to_dict` used `Any` type. Fixed with `FrameworkReport`.

106 Phase 6 tests written:
- `test_unified.py` (12) — Empty/single/multi framework, totals, compliance rate, flags, exception handling
- `test_gap_analyzer.py` (15) — Gaps, sorting, priority, effort, OWASP, multi-framework, exceptions, category completeness
- `test_markdown.py` (10) — Sections, icons, evidence, recommendations, tables, empty
- `test_json_fmt.py` (10) — Valid JSON, fields, conditional inclusion, counts, empty
- `test_html.py` (12) — Valid HTML, badges, escaping (5 chars + single quotes), self-contained, empty
- `test_summary.py` (11) — Progress bar boundaries, print output
- `test_qa_edge_cases.py` (26) — Category mapping completeness (cross-references requirements.py), unicode, boundary inputs, cross-module integration roundtrips, HTML escaping
- `test_cli.py` (10 added) — Report markdown/json/html, custom output, gaps, verify exit codes


## Quick start

```bash
cd your-project
licit init
licit trace --stats
licit report                              # Compliance report (Markdown)
licit report --format html -o report.html # HTML version
licit gaps                                # What's missing?
licit verify                              # CI/CD gate
```
